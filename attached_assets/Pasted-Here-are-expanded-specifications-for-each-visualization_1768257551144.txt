- Group suggestions by original pair
- Identify consensus (same code suggested by multiple reviewers)
- Surface unanimous suggestions for easy acceptance
- Flag conflicting suggestions for expert discussion

- Total suggestions / unique pairs with suggestions

## Additional Dashboard Considerations

### Global Filters
All visualizations should support filtering by:
- Campaign (single or multi-select)
- Date range
- Reviewer (for admin view)
- Pair type (questionnaire_match vs. loinc_mapping)

### Export Options
Each visualization should have:
- PNG/SVG export for charts
- CSV export for underlying data
- Scheduled email reports (optional future feature)

### Real-time Updates
Consider WebSocket or polling for:
- Live vote counts during active review sessions
- Campaign completion notifications
